Uploading firmware - technical documentation

Details
Category: Radio basics (XRF, URF, ERF & SRF)
Last Updated on Monday, 30 November -0001 00:00

Early versions (prior to a bootloader)
The device will reset normally at all times. New firmware is loaded into the upper 16k and copied into the lower 16k on a commit.

Bootloader 2, 3, 3d
If you issue an ATPG to a device the following happens
1) XRF, ERF and URF (V0.34 onwards):
a. Upon receiving ATPG the device resets
i. If nothing else happens then the device starts up normally running any loaded firmware
ii. If a ~ character is received on the serial line during the first (about 130mS) then the device enters the bootloader, a reset at this point will restart the device normally.
iii. If programming is then started the firmware is erased. At this point if programming is not completed then the device has no firmware and so remains in bootloader mode even after a reset
iv. If there is no valid firmware loaded then the device will startup in bootloader mode
b. URF will only accept ATPG over the serial port – it will return ERR if received over the USB port.
2) URF (V0.33 and prior)
a. Upon receiving ATPG via the USB port the device invalidates firmware and waits for an upload on the Serial port. A reset here will not alter this and firmware must be uploaded for normal operation to resume.


bootloader protocol (-> = tx to XRF, <- = rx from XRF)
->~ (within 130mS of reset - enter bootloader mode)
->Y (request version)
<-3 (version 3)
->S (request sub version)
<-d (sub version d)
->W (start write mode) (or V for Verify)
<-W (or V)(acknowledge and erase flash if writing) 

For each line from the file (xxx.bin)
<-R (ready for next line)
->Send line
<-A (meaning line is good)

For the final line
<-R (ready for next line)
->send final line
<-A (line is good)
<-R (ready for next line)
<-y (processed the final line and upload is good)
Can verify here if needed
->X (exit bootloader) (this is the equivalent to commit)
Device restarts with new firmware

if a N character is received during upload or verify then this signifies an error, you can retry the line - or reset the device and start again.

If a n character is received then this indicates an unrecoverable error - reset the device and try again.
